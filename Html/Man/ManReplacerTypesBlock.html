1.1.                   REPLACER TYPES

 

Replacers can have different types, depending on what kind of data is stored in the field. This are the possible data types:

 

Integer

The value is a whole number.

When the flags indicate any lookup type, the criteria from will be the scope it is referring to. When the flags indicate a standard kind of replacer (NONE, READ_ONLY, CONST), the criteria from and to can be set with the min and max value possible.

 

Numeric

The value is a number, but it can have fractions.

The value of the criteria from and criteria to can contain the min and max value that can be used in this field.

 

String

The value is a free text field.

The criteria from and to can contain the minumum and maximum length of the field. When only the criteria from is specified, it can contain a code that indicates what kind of string it is, for example "email"

Possible codes are:

- ExtendedNumber: a number with point or some additions tokens.

- HouseNumber: a valid house number

-Tel: a valid telephone number

- EMail: a valid email address

- URL: a valid url

- Image: a valid image on disk (on server)

- UrlKey: a valid key for using as part of the url

- NumberList: A list of whole integer numbers

- BlockName: A valid name of a block.

- DomainName: A valid domain name

- Folder: A existing folder on the server.

- IPAddress: A valid IP address

- SubDomainName: A subdomainname

 

Date

This indicates that this field is a date. It can also indicates a date including time, but in most cases there is just a date component used.

The criteria from and to can contain the min and max date that are allowed.

Normaly replacers with this type can have a parameter to format the date. The formatting uses a string. When no formating is given, it uses the defined formatting of the storeview.

Formatting string codes:

- yyyy: 4 digit year

- yy: 2 digit year

-mmmm: full name of the month

- mmm: 3 letter month abbreviation

- mm: 2 digit month number (like 12 or 04)

- m: 1 or 2 digit month (like 12 or 4)

- dd: 2 digit day (like 07 or 30)

- d: 1 or 2 digit day (like 7 or 30)

- hh: 2 digit hour (24 hour based)

-h: 1 / 2 digit hour (24 hour based)

- nn: 2 digit minute

- n: 1 or 2 digit minute

- ss: 2 digit seconds

- s: 1 or 2 digit seconds

All other codes are constants, and placed directly in the output.

Sample: "dd-mm-yyyy"

 

 

TimeSpan

This indicates a certain time difference between two events.

The criteria from and to or not filled.

Normaly replacers with this type can have a parameter to format the time span. The formatting uses a string. When no formating is given, it uses the default formatting "dd:mm:nn:ss".

Formatting string codes:

- dd: 2 or more digit number of days (like 07 or 30, 137)

- d: 1 or more digit number of days (like 7 or 30, 137)

- hh: 2 digit number of hours (24 hour based)

-h: 1 / 2 digit number of hours (24 hour based)

- nn: 2 digit number of minutes

- n: 1 or 2 digit number of minutes

- ss: 2 digit number of seconds

- s: 1 or 2 digit number of seconds

All other codes are constants, and placed directly in the output.

Sample: "dd:hh:mm:ss"

 

StoreViewString

The value is a free text field. However it is possible that each storeview has a different value. This text can be entered for each language.

The criteria from and to can contain the minimum and maximum length of the field. When only the criteria from is specified, it can contain a code that indicates what kind of string it is, for example "email"

 

List

This is indicates that there is actually a list of values. Normally this is a list of a reference to another scope. In the criteria from, you can find the scope it is referring to.

 

Map

This is indicates that there is actually a indexed list of values. Normally this is a list of a reference to another scope. In the criteria from, you can find the scope it is referring to.

 

Reference

This indicates that this field is a single reference to another scope. The criteria from contains the name of that other scope.

 

Enum

This field is a enumeration type. It can have only certain values. Which values it can have depend on the type of enumeration. The enumeration types are listed in this manual. The scope from contains the name of the enumeration type.

 

Boolean

This field can only have two values, yes or no. The output is always: yes/no. When settings also 0/1, true/false can be used.

 

StoreViewBool

This field is a boolean value, but it can be different for each storeview (language). The value is always yes/no.

 

StoreViewEnum

This field is a enumeration type. It can have only certain values. However the value can be set for each storeview independently. Which values it can have depend on the type of enumeration. The enumeration types are listed in this manual. The scope from contains the name of the enumeration type.

 

Void

This field has no value. it is always empty.

 

Amount

This field is a amount of money.

This value can be formated, but if no format is given the default format for that currency (CurrencyUsed) is taken.

The format parameter can be:

"unit": Only the unit (currency unit) is display, not the value itself.

"Plain": The value is shown plain, ready for exporting. No unit is given, no thousand separators, and the decimal separator is a dot (.).

"no-unit" The value is given without unit displayed.

"TDNZ" This is the (optional) second parameter if "no-unit" is specified, otherwise the first parameter. It describes how the separators are and the number of decimals. In this code the letters stand for:

- T: thousand separator character

- D: decimal separator character

- N: number of decimals. (character from 0 to 9)

- Z: zero at decimals. When the decimal are zero use this character.

 

 

Color

This field indicates that the data is a color.

Colors can be formatted using a parameter. The parameter can have the following values:

- "hex": Hexadecimal value preceded by a #. For example #FF0000

- "dec": Decimal value. For example: 16711680

- "name": Name of the color. For example: Red. When the color does not have a html name, the hexadecimal value is displayed.
