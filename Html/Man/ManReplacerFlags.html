15.1.              REPLACERS FLAGS

 

We have the following flags available:

 

NONE:

This is a normal replacer. It has a simple value with given type.

* It can be read using the syntax \[scope.code\].

* It can be set.

* This replacer can have all data types except: List, Map and Reference.

* In the criteria field from and to, there can be a value depending on the datatype of the replacer.

 

ID_FIELD:

This flag indicates that this field is the id of the record. It is always an Integer value.

* Id's can not be set.

* It can be retrieved with [scope.id].

* This replacer can only have the data type: Integer.

* The criteria from and to are not filled.

 

READ_ONLY:

This is a normal replacer for reading. It can be retrieved, however it is not possible to set this value. This value is a calculated value or a related value, that is not directly stored in data tables. It depends on other fields.

* It can be read using the syntax \[scope.code\].

* It can not be set.

* This replacer can have all data types except: List, Map and Reference.

* In the criteria field from and to, there can be a value depending on the datatype of the replacer.

 

OWNED:

This is a replacer that indicates a parent child relation. A record can have a child table with 1 or multiple records of another type. It is also possible that the child table has no records. In case of a reference there is just one record possible, not multiple. In the case of Map or List there can be any number of records (including 0 and 1).

* This can only be used with the looped replacer definition.

* This value can be accessed like: \[scope.code=\[child_scope.some_code\]\]

* This value can not be set.

* The data type of this replacer can only be Map or List or Reference.

* Child records can be added, updated or deleted.

* The criteria from contains the name of the child scope.

 

PARENT:

This replacer indicates that this field is the parent of this scope. It can be used as a reference only. Although all tables do have a parent, this replacer is often not provided, because it has limited use.

* It can be used like: \[scope.code=\[parent_scope.parent_code\]\]

* It is read only, values can not be set.

* The data type is always reference.

* The criteria from contains the name of the parent scope.

 

LOOP:

This replacer indicates that the field can only be looped, and not used as a direct replacer. However it is not owned, so it loops over references to another table.

* It can be used like: \[scope.code=\[other_scope.Other_code\]\]

* It can be set using a comma seperated list of ids.

* It can not be added or deleted.

* It has datatype map or list

* the criteria from contains the scope over which you can loop

 

 

SPECIFY:

This indicates that there is another record related to this record which can be accessed using this replacer. This can be used with the loop type of replacer, however it is not an actual loop, it is a single other record.

* It can be used like: \[scope.code=\[other_scope.Other_code\]\]

* It can be set using a single ID of the other scope

* It has datatype reference or integer

* the criteria from contains the scope it is referring to.

 

LOOKUP:

This indicates that this value is a lookup field to another table. It is always an integer field. This is used to make references to other tables.

* It can be read like \[Scope.code\]

* The data type is always an integer

* It can be set, using a valid ID to the other table

* In some cases it can be set with 0

* The criteria from contains the name of the scope it is referring to.

 

LOOKUP_LIST:

This indicates that there is a multiple reference to a certain table. The field is actually a list of ids, that all point to different records in the other scope.

*It can be read, the value is a comma separated list of Id's.

* It can be read like [Scope.code]

* It can be set, with a comma separated list of valid ids.

* The data type is always string

* The criteria to is the scope the id's are referring to.

 

CONST:

This is a normal replacer that can be read, but it can not be set. A const field is a field can not be set, because of some internal or logical reason. They are real fields in the database though, this in contrast with READ_ONLY fields.

* It can be read using the syntax [scope.code].

* It can not be set.

* This replacer can have all data types except: List, Map and Reference.

* In the criteria field from and to, there can be a value depending on the datatype of the replacer.

 

READ_ONLY_LOOKUP:

This indicates that this value is a lookup field to another table. It is always an integer field. This is used to make references to other tables. The field can not be changed.

* It can be read like [Scope.code]

* The data type is always an integer

* It can not be set

* The criteria from contains the name of the scope it is referring to.

 

READ_ONLY_OWNED:

This is a replacer that indicates a parent child relation. A record can have a child table with 1 or multiple records of another type. It is also possible that the child table has no records. In case of a reference there is just one record possible, not multiple. In the case of Map or List there can be any number of records (including 0 and 1). It can not be changed.

* This can only be used with the looped replacer definition.

* This value can be accessed like: \[scope.code=\[child_scope.some_code\]\]

* This value can not be set.

* It can not be added, deleted or updated

* The criteria from contains the name of the child scope.

 

 

READ_ONLY_LOOP:

This replacer indicates that the field can only be looped, and not used as a direct replacer. However it is not owned, so it loops over references to another table. It can not be changed.

* It can be used like: \[scope.code=\[other_scope.Other_code\]\]

* It cannot be set.

* It can not be added or deleted.

* It has datatype map or list

* the criteria from contains the scope over which you can loop

 

READ_ONLY_SPECIFY:

This indicates that there is another record related to this record which can be accessed using this replacer. This can be used with the loop type of replacer, however it is not an actual loop, it is a single other record.

* It can be used like: \[scope.code=\[other_scope.Other_code\]\]

* It can not be set.

* It has datatype reference or integer

* the criteria from contains the scope it is referring to.
