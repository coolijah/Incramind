15.1.              REPLACERS

 

Replacers are the codes that can be used in the html, that will be replaced by the value from the database of the shop. Replacers are recognizable by the use of a pair of square braces [ and ].

For example: "<p>This is: \[Website.Name\].</p>", will substitute the name of the website between the square braces. If your shop is called "my first shop", then the html above will result in: "<p>This is: my first shop.</p>".

There are replacers for every piece of data in the web shop, and some replacers can be more complex, like conditional or looping over certain values. For more information see the general explanation about replacers.

To know what replacers there are available we can list the replacers per subject, or find the right replacer.

 

 

15.2.              SCOPE

 

Scope is the grouping of the replacers. This is mainly done by the table in which the data of the replacer is placed. For example the price of a product is represented as \[Product.Price\]. Here is price the scope. A replacer always has a scope and name, and combined forming the replacer. The scope is mostly the same as a table, but in some cases there is a difference. For example we have the scope "Customer" and "CurrentCustomer". The last scope always represents the current customer that is logged in, and the customer scope can be any customer, when looping over the customer, etc.

There is a list of scopes available in the manual, and when you click on the scope, the relationship with other scopes will show up, and all the replacers of that scope are listed.

 

 

15.3.              TABLES

 

Tables are the available data tables of IncraShop. When you select an overview of the tables you will see a graph with the table names and their relationship. That is the relationship of the data. For example there is a table called "Product". The parent of that table is "Website". This means that you can have multiple products for a website. The table "Product" also has child tables, for example "ProductImage". A product can have more than one image.

This shows how the data is ordered. It is important to understand a bit about the ordering of the data, because when making the html code we can loop child tables and need to know what scope/table we are dealing with.

 

 

15.4.              FILTERS

 

Filters are used in the RestApi to retrieve the data. They are also used in the html code, but mostly for admin related pages. So the average designer will have not much to do with filters.

The names of the filters are often similar to the name of the tables and will then retrieve the data of the corresponding table. There are however some filters that do not correspond directly to the table. For example the filter "CategoryProducts" will get all the products that are in a certain category. There is no seperate table for that. But it will loop the products.

The list of filters in the reference manual shows what is the parent of the filter, over what table it is looping over, and a description.

 

 

15.5.              MODULES

 

Modules are the building blocks of IncraShop. For every major functionality there is a module.

Mostly the modules use the name of the module in the first part of the url. This can be edited. Sometimes also the url can be edited directly. For example the module "product" will not use the url product, but every product can set its own url-key.

The list of modules will have a description per module, and the sub-urls that are used in this module, and the blocks that are invoked.

Modules are also called models sometimes.

 

 

15.6.              URLS

 

This will list all the urls (addresses in the browser) that IncraShop supports. This is basically all the available functionality. it will list what the url does, what parameters there are, and how the url is build up. It It will also list the html blocks that are called when using this url, if any.

The urls can be listed per module, or as a total list.

 

 

 

15.7.              BLOCKS

 

The Blocks are the html blocks that are called by the program when the user navigates to a certain url. This rml blocks contain the first code to be executed when that url is accessed. Of course in any design there are many other blocks that are called from that first block. These are not listed, because they depend on the design that the user himself have made.

The blocks can be listed per module or all.

 

 

15.8.              TYPES

 

There are different type of replacers. Some represents numeric values, others text values (called strings). There are also types that can have only certain values, and we call then enumerations, or for short enums. The value is limited by a value from a list of possible values.

In the types these enumerations are listed, with the meaning of each value.
